# Quick Start

## 1. Install

```bash
bun add -g @bnhpio/thyme-cli
thyme --version
```

Keep the CLI global and the SDK local so projects share a single toolchain.

## 2. Create a Workspace

```bash
thyme create treasury-automation
cd treasury-automation
bun install
cp .env.example .env
```

Populate `.env` immediately—never paste raw private keys into code; use environment references and `bun run` scripts to hydrate secrets.

## 3. Generate a Function

```bash
thyme create stream-payout
```

You now have:

```
functions/
└─ stream-payout/
   ├─ index.ts    # runner (imports @bnhpio/thyme-sdk)
   ├─ schema.ts   # Zod schema
   └─ args.json   # sample payload
```

## 4. Authenticate Once

```bash
thyme auth <token-from-console> --env .env
```

The CLI injects `THYME_AUTH_TOKEN` into your `.env`, keeping the credential outside git history (ensure `.env` stays in `.gitignore`).

## 5. Simulate and Upload

```bash
thyme simulate stream-payout --profile sepolia-test
thyme upload stream-payout --organizationId treasury
```

Simulations block on schema validation, sandbox execution, and viem’s `simulateCalls`. Upload reuses the compiled artifacts, compresses them, and posts to `DEPLOY_URL`.

## 6. Automate

Add CI hooks that call:

```bash
bun test
thyme simulate stream-payout --profile sepolia-test --env .env.ci
```

Block releases unless simulations pass—deployments (`thyme upload`) should only run in release pipelines guarded by limited-scope auth tokens.
