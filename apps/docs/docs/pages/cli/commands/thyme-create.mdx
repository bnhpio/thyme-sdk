# `thyme create`

Bootstrap a Thyme workspace or add a function module. The command auto-detects whether you are inside an existing project by inspecting `untl.toml`.

## Usage

```bash
# New project (run outside an existing workspace)
thyme create treasury-automation --path ./automations

# New function (run inside an existing workspace)
thyme create stream-payout --template default
```

## Options

| Flag | Description | Default |
| --- | --- | --- |
| `--path`, `-p` | Destination directory for the project/function | `./` |
| `--template`, `-t` | Template alias (`default`, more coming) | `default` |

## Execution Flow

1. Resolve working directory and check for `untl.toml`
2. If missing, generate a workspace (package.json, `tsconfig.json`, `.env.example`, `functions/`)
3. Create or reuse `functions/<name>/`
4. Copy runner/schema/args templates (TypeScript + Zod)
5. Print next steps (`bun install`, `cp .env.example .env`)

## Resulting Structure

```
treasury-automation/
├─ package.json
├─ tsconfig.json
├─ untl.toml
├─ .env.example
└─ functions/
   └─ stream-payout/
      ├─ index.ts
      ├─ schema.ts
      └─ args.json
```

## Tips

- Keep shared helpers (ERC20 ABIs, math utils) in `packages/` or `lib/` instead of duplicating them inside each function.
- Use descriptive names (`stream-payout`, `hedge-monitor`) so CI reports remain readable.
- After scaffolding, run `bun install` once to pull SDK dependencies referenced in generated files.
