# `thyme auth`

Persist an authentication token once so subsequent commands (especially `thyme upload`) can reuse it automatically.

## Usage

```bash
thyme auth <token> --env .env.local
```

## Options

| Flag | Description | Default |
| --- | --- | --- |
| `--env <path>` | Target environment file | `.env` |

## Execution Flow

1. Loads or creates the specified `.env`
2. Inserts/updates `THYME_AUTH_TOKEN=<token>`
3. Keeps other variables untouched
4. Prints confirmation along with the file path

## Best Practices

- Treat auth tokens like API keysâ€”never commit them. Verify `.env` is ignored by git.
- Use different env files per environment (`.env`, `.env.staging`, `.env.ci`).
- Rotate tokens regularly and rerun `thyme auth` to propagate new values.
- If you must override the stored token temporarily, pass `--authToken` directly to `thyme upload`; it will take precedence without mutating the `.env`.
